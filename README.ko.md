# libre

**검열할 수 없는 커뮤니티.**

libre는 [Nostr](https://nostr.com) 프로토콜 위에 구축된, 탈중앙화되고 검열 저항성을 가진
커뮤니티 앱(레딧 스타일 포럼)입니다. 압수할 중앙 서버도, 압박할 회사도, 차단할 단일
도메인도 없습니다. 게시물은 전 세계 독립 운영자들이 운영하는 릴레이 위에 존재합니다.
사용자의 신원은 직접 소유하고 관리하는 키입니다.

## Why

점점 더 많은 정부가 발언을 삭제하도록 명령할 수 있는 법을 동원하고 있습니다. 모호하게
정의된 "허위" 또는 "조작된" 정보를 이유로 징벌적 손해배상과 벌금을 부과하고, 게시물
삭제, 신고, 모더레이션 의무를 대형 플랫폼에 떠넘기는 식입니다. 그 기준은 종종 정당한
발언을 침묵시키는 스위치가 될 만큼 느슨하다는 비판을 받습니다.

libre는 이에 대한 직접적인 답입니다. **그 스위치 자체를 존재하지 않게 만드는 것.**
게시물을 내리라고 명령할 대상이 없다면, 중앙 운영자도, 회사도, 병목 지점도 없다면,
끌 스위치 자체가 없는 것입니다.

## How it works

- **콘텐츠는 libre 서버가 아니라 Nostr 릴레이 위에 존재합니다.** libre는 어디에서든
  호스팅하거나 로컬에서 실행할 수 있는 정적 HTML + JavaScript일 뿐입니다.
- **커뮤니티는 NIP-72(모더레이션 커뮤니티)를 사용합니다.** 커뮤니티는 주소 지정 가능한
  이벤트(`kind 34550`)이며, 게시물과 답글은 여기에 속하는 NIP-22 코멘트(`kind 1111`)입니다.
- **사용자가 자신의 키를 소유합니다.** NIP-07 브라우저 확장으로 로그인하거나, 로컬 신원을
  생성하세요. 키는 절대 기기를 벗어나지 않습니다.
- **릴레이는 사용자가 직접 선택합니다.** 여러 관할권에 걸친 기본값과 함께 제공되며, 직접
  추가할 수도 있습니다. 릴레이가 하나라도 살아있는 한 커뮤니티는 존속합니다.

## Design

랜딩 페이지가 아니라 Hacker News에 가깝게 의도적으로 단순합니다. 시스템 폰트, 최소한의
CSS, 프레임워크 없음, 추적 없음, 사용자가 설정한 릴레이와 미디어 호스트를 제외하면
런타임에 서드파티 호출이 전혀 없습니다. 처음부터 다국어를 지원합니다(영어 + 한국어).

## Run it locally

```bash
cd web
npm install          # build-time only: bundles nostr-tools
npm run build:vendor # -> web/vendor/nostr-tools.js (committed)
# serve the static folder with anything:
npx serve .          # or: python3 -m http.server
```

출력된 URL을 열어보세요. 그게 이 앱의 전부입니다. 릴레이와 통신하는 정적 파일입니다.

## Run a relay (grow the network)

독립 릴레이가 많아질수록 검열 저항성도 커집니다. 검열하는 국가 **밖의** 아무 VPS에서:

```bash
cd relay
# edit Caddyfile + strfry.conf with your domain
docker compose up -d
```

그런 다음 libre → **relays**에서 `wss://your-domain`을 추가하세요. 자세한 내용은
[`docs/deploy.md`](docs/deploy.md)를 참고하세요.

## Deploy the client (many doors)

`web/`을 아무 정적 호스트에 올리고 **동시에** IPFS에 핀하고, `.onion` 주소로도 노출하세요.
여러 진입점이 있으면 하나가 막혀도 접근이 끊기지 않습니다. 자세한 내용은
[`docs/deploy.md`](docs/deploy.md)와 [`docs/threat-model.md`](docs/threat-model.md)를
참고하세요.

## 웹사이트가 차단되면

웹사이트가 차단되어도 libre 자체가 사라지는 것은 아닙니다. 이 앱은 그냥 파일 몇 개일
뿐입니다. 빌드된 단일 파일 `index.html`(또는 프로젝트 ZIP)의 사본을 안전한 곳에 하나
보관해 두면, 언제든 내 컴퓨터에서 열 수 있습니다. 웹사이트 없이도 릴레이에 그대로
연결되어 계속 작동합니다. 비개발자도 따라 할 수 있는 단계별 안내이며, 휴대폰에서 여는
방법도 포함되어 있습니다:
[내 컴퓨터에서 직접 libre 실행하기](docs/deploy.ko.md#웹사이트가-차단되면-내-컴퓨터에서-직접-libre-실행하기).
