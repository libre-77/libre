# Threat model

libre는 **합법적인 발언을 검열로부터 지키는** 것을 목표로 합니다. 그 자체로 익명성
도구는 아니며, 전 세계적 수동 공격자(global passive adversary)를 막아내지 못합니다.
이 한계를 알아두세요.

## Adversary

자신의 관할권 내에서 다음을 할 수 있는 국가 또는 ISP 수준의 행위자:

- 회사/운영자에게 콘텐츠 삭제나 데이터 제공을 강제할 수 있고,
- 서버와 도메인을 압수하고, DNS/SNI/IP 차단을 명령할 수 있고,
- 앱스토어에 목록 삭제를 압박할 수 있고,
- 실명/전화번호 인증을 요구할 수 있고,
- 네트워크 트래픽을 관찰하고 IP 주소를 식별할 수 있는 존재입니다.

## Design responses

| Adversary capability | libre response |
| --- | --- |
| 중앙 서버를 압수/통제 | 애초에 중앙 서버가 없습니다. 콘텐츠는 독립적인 Nostr 릴레이 위에 있고, 클라이언트는 정적 파일입니다. |
| 도메인/DNS/SNI 차단 | 단일 도메인에 의존하지 않습니다. 클라이언트는 여러 도메인과 IPFS, `.onion`에 미러링되어 있고, 릴레이는 사용자가 직접 편집 가능한 다중 엔드포인트 목록입니다. |
| 앱스토어 목록 삭제 | 스토어를 통해서만이 아니라 PWA와 직접 배포 APK, F-Droid로도 배포합니다. |
| 실명/전화번호 인증 의무화 | 계정 자체가 없습니다. 신원은 암호학적 키이며, 전화번호도 이메일도 인증 절차도 없습니다. |
| 게시물 삭제 명령 | 명령을 따를 수 있는 중앙 주체가 없습니다. 모더레이션은 전역이 아니라 커뮤니티별(NIP-72)이고 클라이언트별로 이루어집니다. |
| 트래픽 분석 / IP 신원 노출 | libre는 Tor/VPN 위에서도 동작합니다. 네트워크 차원의 익명성이 필요한 사용자는 이를 직접 사용해야 합니다(아래 참고). |

## What libre does NOT protect against

- **네트워크 수준의 신원 노출.** 릴레이는 접속하는 IP를 볼 수 있습니다. *libre를
  사용한다는 사실* 자체나 *어디서 게시하는지*를 숨겨야 한다면 Tor나 신뢰할 수 있는
  VPN을 통해 접속하세요. libre는 트래픽을 대신 라우팅해주지 않습니다.
- **키 유출.** 개인 키를 가진 사람은 곧 당신입니다. NIP-07 확장이나 하드웨어 서명기가
  로컬 키 대체 방식보다 안전합니다.
- **콘텐츠 수준의 익명성 해제.** 프로토콜과 무관하게 문체, 게시 시각, 스스로 드러낸
  정보로 신원이 특정될 수 있습니다.
- **게시물에 대한 법적 책임.** 검열 저항성이 면책을 뜻하지는 않습니다. 명예훼손이나
  불법 콘텐츠는 어디에 있든 여전히 불법입니다.
- **로그를 남기는 릴레이.** 릴레이는 데이터를 보관할 수 있습니다. 게시하는 모든 것이
  공개되고 영구적으로 남는다고 가정하세요.

## Operator note

검열하는 국가 내부에서 클라이언트나 릴레이를 운영하면, 인프라가 해외에 있더라도
운영자 개인이 노출될 수 있습니다. 가능하면 관할권 밖에서 호스팅하고, 익명으로 운영 및
결제하는 방안을 고려하세요. 현지 법률 자문을 받으세요.
